## Методология тестирования

Методы сравнения:

Рекурсивный (без мемоизации)

Итеративный (без мемоизации)

Рекурсивный с мемоизацией

Итеративный с мемоизацией

## Ключевые результаты
1. Производительность по методам
Без мемоизации:

Итеративный метод стабильно быстрее для всех значений n

Рекурсивный метод медленнее из-за накладных расходов на вызовы функций

Разница становится более заметной при больших n

С мемоизацией:

Первый вызов: сравним по времени с немемоизированными версиями

Повторные вызовы: практически мгновенные (из кеша)

Мемоизация добавляет небольшие накладные расходы при первом вызове

2. Анализ временной сложности
Итеративный: O(n) по времени, O(1) по памяти

Рекурсивный: O(n) по времени, O(n) по памяти (стек вызовов)

С мемоизацией: O(1) для повторных вызовов одинаковых n

3. Ограничения
Рекурсивный метод: ограничен максимальной глубиной рекурсии

Большие числа: проблемы с вычислением из-за переполнения целых чисел

## Выводы
Для одиночных вычислений итеративный метод предпочтительнее

Для многократных вычислений с повторяющимися значениями мемоизация значительно ускоряет работу

Рекурсивный подход проще в реализации, но менее эффективен

Мемоизация наиболее полезна когда:

Функция вызывается многократно с одинаковыми аргументами

Вычисления сложные и требуют много времени

Доступно достаточно памяти для хранения кеша

Избегайте глубокой рекурсии для больших n из-за ограничений стека
